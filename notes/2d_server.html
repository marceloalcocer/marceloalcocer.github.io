WAMP Stack Installation¶

Setting up MediaWiki on own server requires:

  1. Web server
  2. PHP installation
  3. Database installation

Easiest way of getting all these is a WAMP/LAMP/WIMP/LIMP stack (Windows|Linux, Apache|ISS, MySQL, PHP). Nice bundle which facilities installation and configuration.

Nice WAMP stack for Windows is XAMPP [1]. Added benisfit that it supports Bitnami installer for MediaWiki — installer to simplify installation and configuration of MediWiki alongside WAMP stack.

As such, downloaded and installed XAMPP 7.0.8. Installed components:

  1. Apache (webserver)
  2. MariaDB (database)
  3. PHP
  4. phpMyAdmin (SQL management web interface)
  5. Webalizer (logging)

Start Apache and MariaDB. Can now see default XAMPP start page at http://2d-acq.root.sx.

Securing SQL¶

XAMPP stresses that it is not production ready and has lots of security holes [2] — fine for dev, but not for public release. Latest version of XAMPP however seem to have patched the most glaring holes [3]. Only real thing left to secure is MySQL root password.

Set via XAMPP console using:

mysqladmin --user=root password <PASSWORD>

N.b. quote <PASSWORD>!

Changed phpmyadmin.ini to authenticate using http rather than config file values. phpMyAdmin now asks for password.

Testing access from outside of ChemPhys LAN. No access to either web server from outside (home). Good.

MediaWiki Installation¶

Now installing MediaWiki using Bitnami installer [4].

N.b. application password is password to be used by SQL user bitnami_mediawiki for all MediaWiki access to SQL DB. It’s stored in plaintext in LocalSettings.php so it should be disposable.

Added new user Malcocer with admin privileges during installation.

Mercurial Web Hosting¶

Now want to set up web interface to HG repos. Can do in many ways [5]. Simplest probably hgweb though.

hgweb requires mercurial python libraries however. As mercurial built on Python 2.x, on Windows these are only supplied for Python 2.x. As such did following:

  1. Uninstalled existing Mercurial installation (3.5)
  2. Installed Python 2.7 alongside existing Python 3.4
  3. Made python2 and python3 symlinks to executables
  4. Added Python 2.7 to path (above Python 3.4)
  5. Reinstalled Mercurial (3.9) with Python 2 modules [6]

Now default Python interpreter is 2.7 and can import mercurial module in python.

Proceeding with hgweb installation. Points to note:

  1. Download latest version of hgweb.cgi as not included in mercurial installation(?!)
  2. Set shebang path correctly in hgweb.cgi
  3. Place hgweb.cgi in cgi-bin
  4. Set apache alias in httpd.conf (in included conf file actually)
  5. Denied push access for each repo individually in repo hgrc file

Works beautifully now.

Finally, themed with boundstate theme [7]. Just had to modify Repositories link to point to /hg as per issue #13.

Filezilla FTP Server¶

Setting up FTP server to allow apache htdocs to be modified without needing RDP to 2d-acq. This should be useful for uploading files which are too large/prohibited by MediaWiki.

Configuration [8]:

  1. Run FTP server
  2. Select to allow FTP server through firewall on private networks
  3. Open admin panel
  4. Connect as admin (requests new password on first time logon)
  5. Configure general server settings (all left as default FTM)
  6. Add new user(group) with associated directory/ies access and password

Can now connect using FTP client (e.g. Filezilla Client, Nautilus, etc).

[8]https://wiki.filezilla-project.org/FileZilla_FTP_Server
[9]https://wiki.filezilla-project.org/Network_Configuration
