Git hosted on WAMP¶

Aim: Host Git repository on Apache on Windows (XAMPP WAMP stack)

Initially tried to handle user authentication via FTP access. Could happily push using git ftp, but had no luck pulling as no git server scripts were present on server.

As have to install Git on server, might as well try more standard setup of Git over (smart) HTTP.

HTTP¶

Download Git for Windows official installation [1]. This comes bundled with MinGW installation which provide basic GNU/Linux tools required for running Git.

Follow steps in Git book on setting up Git over Smart HTTP [2]. N.b. Location of main Git http script (git-http-backend) will probably not be correct but rather C:\\Program Files\\Git\\mingw64\\libexec\\git-core\\git-http-backend.exe [3].

[2]https://git-scm.com/book/en/v2/Git-on-the-Server-Smart-HTTP
[3]https://stackoverflow.com/questions/33829145/git-smarthttp-where-is-git-http-backend-exe/33829578

Useful:

  • http://www.jeremyskinner.co.uk/2010/07/31/hosting-a-git-server-under-apache-on-windows/

FTP¶

Git natively supports some operations over FTP protocol, viz. clone and fetch [4]. For further operations (e.g. push) can use git-ftp [5].

Download git-ftp and install:

make install-all

Setup global/user/local git config with server, username, etc. N.b. Server path will be root directory of project, so best to define per repository in local git config file.

Upload repo:

git ftp init -u <USERNAME> -P

Install lftp:

sudo apt-get install lftp
[4]https://stackoverflow.com/questions/4664251/does-git-support-push-to-an-ftp-server
[5]https://github.com/git-ftp/git-ftp
